[Mistakes I see engineers making in their code reviews](https://www.seangoedecke.com/good-code-reviews/)

Ideally five or six comments max in a review

Don't only review the diff - look at other places in the code as well

Most reviews should be approvals (for some kinds of code bases)

Review with a “will this work” filter, not with a “is this exactly how I would have done it” filter
